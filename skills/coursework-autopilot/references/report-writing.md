# Report Writing

Read this file before drafting the final coursework report.

## Writing expectations

- Keep the main body paragraph-first and academically formal.
- Explain not only what was implemented, but why the design was chosen, how the program behaves, and what the results show.
- Use bullet lists only for genuinely list-shaped material such as environment summaries, algorithm names, or file inventories.

## Recommended section behavior

- `实验目的` or `任务概述`
  Explain the assignment background, target problem, and expected outcomes.
- `需求分析`
  Translate the teacher materials into concrete functional and non-functional requirements.
- `总体设计`
  Describe module boundaries, process flow, and core design choices.
- `详细设计与实现`
  Walk through key data structures, algorithms, control flow, and important code paths.
- `实验结果与分析`
  Tie observed outputs back to operating-system concepts, constraints, and tradeoffs.
- `总结与心得`
  Reflect on implementation lessons, pitfalls, and possible improvements in a formal tone.

## Template usage

- Start from `assets/report-template/REPORT.md` when no teacher template is provided.
- If the archive includes a Word template, keep its required headings and formatting constraints.
- Expand the placeholder text into full prose before considering the report done.
